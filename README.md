## How I Work

- **Backend before UI**: I define data models, flows, and API contracts first; UI layers are built on top of a stable core.
- **Processes, not pages**: Every user interaction maps to a clear backend process (create, approve, cancel, retry, etc.).
- **Reliable by default**: Error handling, logging, and predictable responses are part of the first implementation, not phases later.
- **Small, composable services**: Prefer modular controllers and services over monolith files that try to do everything.
- **Docs as contracts**: I treat endpoint documentation and examples as an agreement between backend and any client.
